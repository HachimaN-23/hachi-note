## Vision

A web-based notes app that takes the best ideas from Obsidian — wikilinks, backlinks, daily notes, outline navigation, YAML frontmatter — and makes them available on any device for free. No paid sync. No plugins. Just works.

The differentiator: **free web sync + markdown + structured metadata**. Obsidian charges for sync. This doesn't.

## Scope Decisions

| # | Feature | Effort | Priority | Rationale |
|---|---------|--------|----------|-----------|
| 1 | Wikilinks `[[...]]` | M | P1 | Core differentiator — makes notes interconnected |
| 2 | Backlinks | M | P1 | Completes the wikilink loop — shows who references you |
| 3 | Daily Notes | S | P2 | Daily driver feature — everyone uses this |
| 4 | Outline (heading navigator) | S | P2 | Power user feature — great for long notes |
| 5 | YAML Frontmatter | S | P3 | Structured metadata — makes notes feel organized |

## Accepted Scope

- Wikilinks: `[[Note Title]]` syntax in markdown, renders as clickable link to that note
- Backlinks: When viewing a note, show which other notes link to it via wikilinks
- Daily Notes: "Today" button creates/opens a note titled with today's date (YYYY-MM-DD)
- Outline: Sidebar component showing H1/H2/H3 headings, click to jump to section
- YAML Frontmatter: Parse `---` blocks at note top, extract tags/aliases/custom fields

## NOT in scope

- Graph View — too complex for v1, deferred to v2
- Callouts `> [!info]` — nice but not essential, deferred
- Templates — premature, deferred
- Canvas/Whiteboard — way too much, deferred
- Plugin system — future consideration only

## What already exists

- Markdown rendering (react-markdown) — ✅ reuse directly
- Tags system (DB-backed) — ✅ extend with YAML tag extraction
- Full-text search (FTS5) — ✅ extend to search wikilink targets
- Folders — ✅ no changes needed
- Checklist items — ✅ no changes needed
- API routes — ✅ extend with new endpoints

### Data Flow — Wikilinks

```
USER TYPES [[Note Title]] IN EDITOR
        │
        ▼
NOTE SAVED WITH RAW [[...]] IN CONTENT
        │
        ▼
WHEN RENDERING:
  wikilinks.ts parses content
  finds all [[...]] patterns
  resolves to note IDs (or creates "unresolved" links)
  ReactMarkdown renders as <a> tags
        │
        ▼
USER CLICKS WIKILINK
        │
        ▼
NAVIGATES TO TARGET NOTE
```

### Data Flow — Backlinks

```
USER OPENS NOTE #42
        │
        ▼
API: GET /api/notes/42/backlinks
        │
        ▼
SQL QUERY:
  SELECT * FROM notes
  WHERE content LIKE '%[[Note Title]]%'
  AND id != 42
        │
        ▼
RETURN LIST OF NOTES THAT LINK TO #42
        │
        ▼
BACKLINKS PANEL RENDERS IN NOTE VIEW
```

### Data Flow — Daily Notes

```
USER CLICKS "Today" BUTTON
        │
        ▼
API: POST /api/daily-notes
  { date: "2026-06-27" }
        │
        ▼
SQL: Check if note with title "2026-06-27" exists in daily-notes folder
  IF EXISTS → return it
  IF NOT → create it with template:
    ---
    date: 2026-06-27
    type: daily
    ---
    # 2026-06-27\n\n
        │
        ▼
NAVIGATE TO THAT NOTE (open in editor)
```

### Data Flow — Outline

```
NOTE VIEW OPENS
        │
        ▼
PARSE MARKDOWN CONTENT FOR HEADINGS
  regex: /^(#{1,3})\s+(.+)$/gm
        │
        ▼
BUILD HEADING TREE:
  [
    { level: 1, text: "Section A", id: "section-a" },
    { level: 2, text: "Sub A.1", id: "sub-a1" },
    { level: 1, text: "Section B", id: "section-b" }
  ]
        │
        ▼
OUTLINE COMPONENT RENDERS
  - Indented list based on level
  - Click scrolls to heading (via id anchor)
  - Highlights current section based on scroll position
```

### Data Flow — YAML Frontmatter

```
NOTE SAVED WITH CONTENT:
  ---
  tags: [react, nextjs]
  aliases: ["My Note", "Note A"]
  status: draft
  ---
  # Actual content here
        │
        ▼
frontmatter.ts PARSES:
  - Extracts YAML between --- delimiters
  - Returns { metadata: {...}, content: "remaining markdown" }
        │
        ▼
METADATA STORED:
  - tags → merged with existing tag system
  - aliases → used for wikilink resolution
  - custom fields → stored as JSON in new metadata column
```

### Database Changes

```sql
-- New table for daily notes (optional, can reuse folders)

-- Wikilinks: no new table needed, parsed from content at render time
-- Backlinks: computed query, no storage needed

```

## Error & Rescue Map

| Method/Codepath | What Can Go Wrong | Exception Class | Rescued? | User Sees |
|-----------------|-------------------|-----------------|----------|-----------|
| wikilinks.ts: parseWikilinks() | Malformed syntax `[[` without closing `]]` | ParseError | Y | Ignore incomplete links, render as text |
| wikilinks.ts: resolveLink() | Target note doesn't exist | NotFoundError | Y | Render as "unresolved" link (dashed underline) |
| frontmatter.ts: parseFrontmatter() | Invalid YAML syntax | YAMLParseError | Y | Show note content without metadata, log warning |
| frontmatter.ts: parseFrontmatter() | Empty frontmatter block | N/A | Y | Return empty metadata object |
| GET /api/notes/[id]/backlinks | Note not found | NotFoundError | Y | Return empty array |
| POST /api/daily-notes | DB write fails | DatabaseError | Y | Return 500, show toast "Failed to create daily note" |
| Outline: parseHeadings() | Content has no headings | N/A | Y | Show "No sections" message |
| Outline: scrollToHeading() | Heading ID not found in DOM | DOMError | Y | No-op, don't crash |

## Security & Threat Model

| Threat | Likelihood | Impact | Mitigated? |
|--------|-----------|--------|------------|
| Wikilink injection (XSS via `[[<script>]]`) | Low | High | ✅ ReactMarkdown sanitizes HTML. Wikilinks resolved to note titles, not raw input. |
| Frontmatter injection (YAML bomb) | Low | Med | ✅ YAML parser has depth limits. Frontmatter capped at 1KB. |
| Backlink enumeration (info leak) | Low | Low | ⚠️ Backlinks only show note titles, not content. Acceptable for personal use. |
| Daily note title injection | Low | Low | ✅ Title is date-formatted, not user-controlled. |

## Test Plan

| Feature | Happy Path | Edge Case | Failure |
|---------|-----------|-----------|---------|
| Wikilinks | `[[Note]]` renders as link | `[[Nonexistent]]` shows unresolved | `[[` without `]]` renders as text |
| Backlinks | Note A links to B, B shows backlink | Self-link excluded | No notes link to target |
| Daily Notes | Click "Today", note created | Already exists, opens existing | DB error |
| Outline | H1/H2/H3 parsed correctly | Note with no headings | Empty outline |
| Frontmatter | `---` block parsed, metadata extracted | Invalid YAML | Content shown without metadata |
| Frontmatter | Tags in frontmatter merge with tag system | Duplicate tags | Deduped |

## Performance Considerations

- **Wikilinks**: Parsed client-side, no DB hit per link. Resolution happens once on note load.
- **Backlinks**: Single SQL query with LIKE. Fast for <1000 notes. If >5000 notes, add full-text index on content for backlink search.
- **Outline**: Client-side regex parse. Negligible cost.
- **Frontmatter**: Client-side YAML parse. `js-yaml` is ~20KB gzipped. Worth it.
- **Daily Notes**: Single DB lookup. Negligible.
